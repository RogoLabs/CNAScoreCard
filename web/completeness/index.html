üìñ Completeness Methodology

üéØ Analysis Scope

This analysis evaluates CVE records against the official CVE JSON Schema, measuring the presence and utilization of both required and optional fields. We assess over 50 different schema elements including core metadata, affected products, problem types, references, and metrics.

üìä Scoring Algorithm

Completeness scores are calculated using a weighted approach: required fields contribute 70% of the score while optional fields contribute 30%. This reflects the critical importance of required fields while still rewarding comprehensive use of optional schema elements.

üîç Field Categories

  • Core Metadata: CVE ID, assigner information, state, and timestamps
  • Vulnerability Information: Descriptions, affected products, problem types
  • Technical Details: CVSS metrics, CPE identifiers, CWE classifications
  • Actionable Information: References, solutions, workarounds, exploits
  • Enhanced Data: Timeline, credits, source information, taxonomy mappings

üéØ Quality Indicators

Beyond simple presence/absence, our analysis includes quality checks such as English language descriptions, named references, version-specific information, and the use of structured identifiers like CPE and CWE.
